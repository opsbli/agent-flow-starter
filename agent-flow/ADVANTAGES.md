# agent-flow Advantages

`agent-flow` is useful when AI development needs more than a prompt.

## Core Advantages

- Code first: scan the real project before writing requirements, design, or code.
- Controlled routing: Light / Standard / Heavy prevents small tasks from becoming ceremony and risky tasks from becoming improvisation.
- Explicit boundaries: `read_files` and `write_files` make AI edits auditable.
- Human checkpoints: protected areas require approval.
- Evidence-driven closure: `VERIFY.md` and AC Evidence connect requirements to proof.
- Durable memory: reusable terms, pitfalls, and decisions go into files instead of chat.
- Self-evolution: each change can improve templates, knowledge, and gates.

## Best Fit

- Legacy projects with new modules.
- Teams using AI across multiple IDEs.
- Projects where public contracts, permissions, schema, workflow, or frontend/backend alignment matter.
- Teams that want reusable decision and knowledge history.

## What It Avoids

- Direct-to-code for non-trivial changes.
- Chat-only requirements.
- Duplicate abstractions before searching.
- Completion claims without verification.
- Hidden changes in protected areas.
