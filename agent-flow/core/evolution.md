# 自我演进

每次 change 完成后，必须写 `EVOLUTION.md`，回答脚手架是否需要调整。

## 自我推演问题

### 1. 架构推演

- 本次新模块是否改变了模块边界？
- 是否出现新的公共能力候选？
- 是否应该从业务模块抽到项目公共层？
- 是否出现循环依赖或隐性耦合？

### 2. 流程推演

- 本次哪些步骤有价值？
- 哪些步骤只是形式主义？
- 是否应该给某类任务降级？
- 是否应该给某类任务升级为 Heavy？

### 3. 验证推演

- 哪个错误差点漏掉？
- 是否能写成 `agent-flow/scripts/*` gate？
- AC 是否能被机器检查？
- code-drift-check / task-boundary-check 是否需要新增规则？

### 4. 模板推演

- `REQUIREMENT.md` 有没有字段多余？
- `DESIGN.md` 有没有遗漏项目关键约束？
- `TASKS.md` 的 `read_files/write_files` 是否足够约束？
- `REPORT.md` 是否能支撑交接？

## EVOLUTION.md 模板

```md
# Evolution

## 本次发现

## 应升级的规则

## 应新增的知识

## 应新增的验证闸门

## 应调整的模板

## 本次不调整的原因
```

## 自动调整原则

可以直接调整：

- 模板中的必填项。
- 知识库条目。
- 验证脚本中的静态检查。
- GO.md 路由提示。

需要人工确认：

- 改变流程分级。
- 改变架构边界。
- 把业务代码抽成公共模块。
- 修改根级 Agent 规则。
